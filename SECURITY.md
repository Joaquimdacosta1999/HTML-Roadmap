# Security Policy

Thank you for helping keep the **HTML Roadmap** safe and secure for everyone! 🚀

This repository is an **open-source educational project** — a structured learning guide that teaches HTML from beginner fundamentals to advanced topics like semantic markup, forms, accessibility, and more.

Because this is primarily **documentation and learning material** (Markdown files with HTML examples), the security risks are generally low. However, we still take security seriously, especially regarding potential issues in code examples (e.g., unsafe practices that could lead to XSS) or incorrect security-related advice.

## Supported Versions

This project is a living roadmap and does not follow traditional versioning like software releases.

| Version / Branch      | Supported          | Notes                              |
|-----------------------|--------------------|------------------------------------|
| `main` branch         | :white_check_mark: | Actively maintained                |
| Older branches / tags | :x:                | Not supported                      |

We recommend always using the latest content from the `main` branch.

## Reporting a Vulnerability

If you discover a security issue in this repository, we encourage you to report it responsibly so we can address it quickly.

### How to Report
Please report security vulnerabilities by **opening a private issue** or by contacting the maintainer directly:

- **Preferred method**: Open a new issue on GitHub and mark it as **"Security"** (or simply describe the issue clearly).
- **Private / sensitive reports**: You can also email the maintainer at `joaquim.costa1999@gmail.com` (please include "HTML-Roadmap Security" in the subject line).

### What to Include in Your Report
- A clear description of the potential vulnerability
- Steps to reproduce the issue (if applicable)
- Any suggested fix or mitigation
- Your contact information (optional, but helpful for follow-up)

### What to Expect
- You will receive an acknowledgment of your report within **48 hours**.
- We will investigate the issue and keep you updated on our progress.
- If the report is accepted, we will work on a fix and credit you (unless you prefer to stay anonymous).
- If the report is declined, we will explain the reason clearly.

**Note**: This is an educational project maintained by a single person in my spare time. While I aim to respond promptly, please be patient — fixes may take a few days depending on the issue.

## How You Can Help and Contribute

Security is a community effort! Here are ways you can contribute to keeping this roadmap secure and high-quality:

- Review HTML code examples for **security best practices** (e.g., escaping output, proper form handling).
- Suggest improvements to sections related to **web security**, accessibility, or safe coding habits.
- Report any outdated or potentially misleading content.
- Help improve the overall quality by following our [CONTRIBUTING.md](https://github.com/Joaquimdacosta1999/HTML-Roadmap/blob/main/CONTRIBUTING.md) guidelines.

By contributing, you help make this resource safer and more valuable for thousands of learners worldwide.
